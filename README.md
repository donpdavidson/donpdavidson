# Hi, I'm Don Davidson

I design practical AI workflow systems with explicit human review, deterministic evidence, and clear rollout boundaries.

My work focuses on turning useful automation into systems people can inspect, test, and control before external action occurs.

## What I build

- Review-first AI workflows
- Fail-closed approval and allowlist controls
- Human-in-the-loop sales and operations automation
- Synthetic test harnesses and deterministic evidence
- Deployment preflight, configuration drift, and rollback planning

## How I work

1. Define the business output and acceptance criteria.
2. Separate local preparation from external action.
3. Build with synthetic fixtures and explicit boundaries.
4. Test positive, negative, and idempotent paths.
5. Map readiness claims to reproducible evidence.
6. Require human approval before publication or production action.

## Principles

- No client material in public examples
- No hidden sends or production actions
- No readiness claim without cited evidence
- No automation across an unresolved approval boundary
- Synthetic examples before controlled rollout
